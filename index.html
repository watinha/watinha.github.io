Example Apps

  • watinha.com homepage: this is a presentation project, which makes use of different web technologies to improve the user experience in the curriculum. The source code is fully available in Github. This project makes use of Jasmine, as a JavaScript Behavior Driven Development framework; CasperJS, to simulate the browser environment during functional tests; JSHint, to validate the JavaScript source code against JS code standards; Mustache template engine, to generate the webpages from templates; CSS and JS code compression techniques to generate faster webpages with compressed JavaScript and CSS files; performance testing with yslow on the production environment of the application; Halstead, Cyclomatic Complexity and Code-coverage metrics, using grunt-complexity plugin for the complexity metrics and istanbul for the code coverage; and finally, the complete build process of the project is managed through Grunt task manager.
  • aria-check-menus: a Selenium WebDriver implementation which has the goal of automatically identifying Drop-down menu widgets in web applications. This project uses the Selenium WebDriver API, browser mutation observers and DOM attributes to navigate through web applications and identify menu widget implementations.
  • javacmd: boilerplate structure for Java projects. It includes a single Makefile which automatically compile Java source code, generates jar executable files, calculates dependencies based in a single jar directory, runs JUnit tests, generates JPA - Java Persistence API configuration files and generates JEE - Java Enterprise Edition applications structure. The goal of this project is to ease the development of Java projects with no use of IDEs. It is similar to Maven, however much simplier, with no XML configuration files.
  • tabindex-counter: a CasperJS script which collects HTML attributes metrics from webpages. This script was used to investigate how a large amount of websites implement Focus Navigation requirements and whether these implementations are associated to ARIA specific design recommendations.
  • aria-check: a Firefox Addon built on top of the Addon-SDK. The Addon runs a generic test-suite, which searches for Tab Widgets implementations and evaluates whether it implements ARIA specification requirements. The ARIA specification describes the use of semantic markup (such as: roles, aria-labels, among others) and keyboard interaction mechanisms for making JavaScript based interface components accessible.
  • Pyccuracy Accessibility Actions: The main objective of this project is to implement accessibility actions for the pyccuracy, web acceptance testing application. Pyccuracy makes use of natural language expressions to generate user stories and acceptance scenarios that are automatically run in the browser, evaluating the behaviour of all layers of a web application (server and client side-scripting). However pyccuracy still have not implemented actions that can be classified as accessible for disabled users like "tabbing navigation", focus possibility for interactive elements, landmarks and header navigation, among other actions. The goal of implementing accessibility actions for pyccuracy is to be able to execute acceptance tests of web applications functionality, considering disabled users task completion with assistive technologies. These users do no rely on the same visual and interactive mechanisms such as mouse and link colors, for example. So pyccuracy actions must consider these characteristics to guarantee that the functionality of the web application is available for all users. From the development perspective, accessibility actions drive automatic accessibility evaluation tools into a Continuous Integration environment. Making it possible to include accessibility non-functional requirements into the development life-cycle of projects and assisting developers while coding solutions for them.
  • Educational Facilita: another web application that aims to automatically provide users with accessible textual content, through the use of Named Entities Recognition (automatically identifying proper nouns and others entities in the text) and Lexical Elaboration (providing users with simplier versions of the same word) techniques. The application identifies named entities in the text and then it provides the user with additional information about those entities. The additional informations are extracted from Wikipedia. The application also presents synonimous for words that are evaluated as complex or difficult to be understood. Educational Facilita was implemented as a Jetpack extension, and by doing so works as a browser plugin feature.
  • Scale Tabs: improved version of the 50 lines contest - Scale Tabs Jetpack addon. This addon release used the Addon-SDK (improved version and final release of the Jetpack prototype) and presents improved features (Address bar integration) and improved layout. Its source code is available in github.
  • YQL-Accessibility: YQL stands for Yahoo! Query Language, as a standard language for querying web resources. It presents a SQL-like syntax that enable queries to be made on RESTful webservices. YQL-Accessibility implements a set of tables (two for the moment) that allow these queries to be executed against web accessibility evaluation websites. It provides developers a simple way and unique way of identifying and reporting accessibility issues that are raised by automatic evaluation tools available in the Web.
